Privacy Policy

Last updated: July 15, 2026

BookmarkNest is a local-first browser extension for organizing X and Twitter bookmarks.

Data stored locally

BookmarkNest stores imported post text, author name and handle, post and media URLs, import time, tags, folders, notes, saved views, archive state, local delete state, and settings in your browser's extension storage. Import diagnostics may include timestamps, counters, limited post ID samples, extension version, and error messages. Diagnostics do not include bookmark text.

Data sent to services

By default, BookmarkNest does not upload bookmark content, notes, tags, folders, or saved views.

  • When you import or enable auto-sync, BookmarkNest sends authenticated requests to X or Twitter using your existing signed-in session to read your bookmarks and related display metadata.
  • If you enable Cloud Sync, the extension encrypts a backup of your local library before upload. The Cloud Sync service receives the encrypted snapshot, not plaintext bookmark content, notes, tags, folders, saved views, authors, or media URLs.

Public testing and payments

BookmarkNest is currently in public testing. We do not collect payment details or process purchases during testing.

Cloud Sync retention

Cloud Sync testing retains up to five encrypted backup versions and may throttle changed backups to one per minute.

Browser permissions

  • storage stores local settings and test access state.
  • downloads saves exports and local backups.
  • clipboardWrite copies original X links.
  • webRequest, declarativeNetRequest, cookies, and X host access support authenticated bookmark imports from your signed-in X session.
  • alarms supports optional background auto-sync.

Your controls

You can delete individual local records, clear all BookmarkNest data, export a local backup, turn Cloud Sync on or off, and export diagnostics for support.
